35 // Monitors omnibox navigations in order to trigger behaviors that depend on
36 // successful navigations.
37 //
38 // Currently three such behaviors exist:
39 // (1) For single-word queries where we can't tell if the entry was a search or
40 // an intranet hostname, the omnibox opens as a search by default, but this
41 // class attempts to open as a URL via an HTTP HEAD request. If successful,
42 // displays an infobar once the search result has also loaded. See
43 // AlternateNavInfoBarDelegate.
44 // (2) Omnibox navigations that complete successfully are added to the
45 // Shortcuts backend.
46 // (3) Omnibox searches that result in a 404 for an auto-generated custom
47 // search engine cause the custom search engine to be deleted.
48 //
49 // Please see the class comment on the base class for important information
50 // about the memory management of this object.
