24 // Monitors omnibox navigations in order to trigger behaviors that depend on
25 // successful navigations.
26 //
27 // Currently three such behaviors exist:
28 // (1) For single-word queries where we can't tell if the entry was a search or
29 // an intranet hostname, the omnibox opens as a search by default, but this
30 // class attempts to open as a URL via an HTTP HEAD request. If successful,
31 // displays an infobar once the search result has also loaded. See
32 // AlternateNavInfoBarDelegate.
33 // (2) Omnibox navigations that complete successfully are added to the
34 // Shortcuts backend.
35 // (3) Omnibox searches that result in a 404 for an auto-generated custom
36 // search engine cause the custom search engine to be deleted.
37 //
38 // Please see the class comment on the base class for important information
39 // about the memory management of this object.
