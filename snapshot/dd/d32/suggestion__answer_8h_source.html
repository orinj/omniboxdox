26 // Structured representation of the JSON payload of a suggestion with an answer.
27 // An answer has exactly two image lines, so called because they are a
28 // combination of text and an optional image URL. Each image line has 1 or more
29 // text fields, each of which is required to contain a string and an integer
30 // type. The text fields are contained in a non-empty vector and two optional
31 // named properties, referred to as "additional text" and "status text".
32 //
33 // When represented in the UI, these elements should be styled and laid out
34 // according to the specification at https://goto.google.com/ais_api.
35 class SuggestionAnswer {
36  public:
37  class TextField;
38  typedef std::vector<TextField> TextFields;
39  typedef std::vector<GURL> URLs;

100  // The above TextType values match what is sent by server, but are not used
101  // normally by new answers. These enum values are used instead, styling
102  // answer text through a client-side process of interpretation that depends
103  // on answer type, text type, and even line rank (first|second). This
104  // interpretation process happens during answer parsing and allows
105  // downstream logic to remain simple, deciding how to present answers
106  // based on a finite set of text types instead of answer properties and rules.
107  // Performance is also improved by doing this once at parse time instead of
108  // every time render text is invalidated.
109  //
110  // A Java counterpart will be generated for this enum.
111  // GENERATED_JAVA_ENUM_PACKAGE: org.chromium.components.omnibox
112  // GENERATED_JAVA_CLASS_NAME_OVERRIDE: AnswerTextStyle
113  enum class TextStyle {
114  NONE = 0,
115  NORMAL,
116  NORMAL_DIM,
117  SECONDARY,
118  BOLD,
119  POSITIVE,
120  NEGATIVE,
121  SUPERIOR, // This is not superscript (see gfx::BaselineStyle).
122  };
