13 // Truncates the clipboard text returned in order to improve performance and
14 // prevent unresponsiveness. For reference, a book is about ~500k characters and
15 // data URLs served by google images are usually 30k characters or less.
16 // We don't use url::kMaxURLChars (2M), as it's too large; it adds 2s+ delays
17 // when right clicking the omnibox for clipboards larger than 2M. Additionally,
18 // a 500k limit also allows us to not have to worry about length when
19 // classifying the text: OmniboxViewViews::GetLabelForCommandId and
20 // OmniboxEditModel::CanPasteAndGo. If we used a larger limit here (e.g. 2M),
21 // then we'd need to limit the text to ~500k later anyways when classifying,
22 // because classifying text longer than 500k adds a ~1s delays.
23 static const size_t kMaxClipboardTextLength = 500 * 1024;
