31 // Autocomplete provider for keyword input.
32 //
33 // After construction, the autocomplete controller repeatedly calls Start()
34 // with some user input, each time expecting to receive a small set of the best
35 // matches (either synchronously or asynchronously).
36 //
37 // To construct these matches, the provider treats user input as a series of
38 // whitespace-delimited tokens and tries to match the first token as the prefix
39 // of a known "keyword". A keyword is some string that maps to a search query
40 // URL; the rest of the user's input is taken as the input to the query. For
41 // example, the keyword "bug" might map to the URL "http://b/issue?id=%s", so
42 // input like "bug 123" would become "http://b/issue?id=123".
43 //
44 // Because we do prefix matching, user input could match more than one keyword
45 // at once. (Example: the input "f jazz" matches all keywords starting with
46 // "f".) We return the best matches, up to three.
47 //
48 // The resulting matches are shown with content specified by the keyword
49 // (usually "Search [name] for %s"), description "(Keyword: [keyword])", and
50 // action "[keyword] %s". If the user has typed a (possibly partial) keyword
51 // but no search terms, the suggested result is shown greyed out, with
52 // "<enter term(s)>" as the substituted input, and does nothing when selected.
