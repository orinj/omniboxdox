60  // Returns an AutocompleteMatch with the given |autocomplete_provider|
61  // for the search |suggestion|, which represents a search via |template_url|.
62  // If |template_url| is NULL, returns a match with an invalid destination URL.
63  //
64  // |input| is the original user input. Text in the input is used to highlight
65  // portions of the match contents to distinguish locally-typed text from
66  // suggested text.
67  //
68  // |input| is also necessary for various other details, like whether we should
69  // allow inline autocompletion and what the transition type should be.
70  // |in_keyword_mode| helps guarantee a non-keyword suggestion does not
71  // appear as the default match when the user is in keyword mode.
72  // |accepted_suggestion| is used to generate Assisted Query Stats.
73  // |append_extra_query_params_from_command_line| should be set if
74  // |template_url| is the default search engine, so the destination URL will
75  // contain any command-line-specified query params.

157  // Callers should pass |sending_search_terms| as true if user input is being
158  // sent along with the |current_page_url|.
159  //
160  // Returns whether we can send the URL of the current page in any suggest
161  // requests. Doing this requires that all the following hold:
162  // * The suggest request is sent over HTTPS. This avoids leaking the current
163  // page URL or personal data in unencrypted network traffic.
164  // * The user has suggest enabled in their settings and is not in incognito
165  // mode. (Incognito disables suggest entirely.)
166  // * The user's suggest provider is Google. We might want to allow other
167  // providers to see this data someday, but for now this has only been
168  // implemented for Google.
169  // * The user is not on the NTP.
170  // * The current URL is HTTP or HTTPS.
171  // * Either one of:
172  // * The user consented to sending URLs of current page to Google and have
173  // them associated with their Google account.
174  // * The suggest endpoint and current page URL are same-origin and
175  // |sending_search_terms| is false. Same-origin suggest endpoints could
176  // have already logged the current page URL when the user accessed it, but
177  // Chrome still shouldn't leak the association between typed search terms
178  // and which tab the user is looking at. On-focus suggest requests never
179  // send search terms.
