28  // Computes the maximum width, in pixels, that can be allocated for the two
29  // parts of an autocomplete result, i.e. the contents and the description.
30  //
31  // When |description_on_separate_line| is true, the caller will be displaying
32  // two separate lines of text, so both contents and description can take up
33  // the full available width. Otherwise, the contents and description are
34  // assumed to be on the same line, with a separator between them.
35  //
36  // When |allow_shrinking_contents| is true, and the contents and description
37  // are together on a line without enough space for both, the code tries to
38  // divide the available space equally between the two, unless this would make
39  // one or both too narrow. Otherwise, the contents is given as much space as
40  // it wants and the description gets the remainder.
