12 // On device head suggest feature uses an on device model which encodes some
13 // top queries into a radix tree (https://en.wikipedia.org/wiki/Radix_tree), to
14 // help users quickly get head suggestions when they are under poor network
15 // condition. When serving, it performs a search in the tree similar as BFS but
16 // only keeping children with high scores, to find top N queries which match
17 // the given prefix.
18 //
19 // Each node in the tree is encoded using following format to optimize storage
20 // (see on_device_head_model_unittest.cc for an example tree model):
21 // ------------------------------------------------------------------------
22 // | max_score_as_root | child_0 | child_1 | ... | child_n-1 | 0 (1 byte) |
23 // ------------------------------------------------------------------------
24 //
25 // Usage of each block in the node:
26 // 1) Block max_score_as_root at the beginning of each node contains the
27 // maximum leaf score can be found in its subtree, which is used for pruning
28 // during tree traversal to improve the search performance: for example,
29 // imagining we have already visited some nodes, sorted them based on their
30 // scores and saved some of them in a structure; now we meet a node with higher
31 // max_score_as_root, since we know we should only show users top N suggestions
32 // with highest scores, we can quickly determine whether we can discard some
33 // node with lower max_score_as_root without physically visiting any of its
34 // children, as none of the children has a score higher than this low
35 // max_score_as_root.
36 // This block has following format:
37 // --------------------------------------
38 // | 1 (1 bit) | score_max | leaf_score |
39 // --------------------------------------
40 // OR
41 // -------------------------
42 // | 0 (1 bit) | score_max |
44 // 1-bit indicator: whether there is a leaf_score at the end of this block.
45 // score_max: the maximum leaf_score can be found if using current node as
46 // root.
47 // leaf_score: only exists when indicator is 1; it is the score of some
48 // complete suggestion ends at current node.
49 //
50 // 2) Block child_i (0 <= i <= n-1) has following format:
51 // -------------------------------------------------------------
52 // | length of text (1 byte) | text | 1 | address of next node |
53 // -------------------------------------------------------------
54 // OR
55 // ---------------------------------------------------
56 // | length of text (1 byte) | text | 0 | leaf_score |
57 // ---------------------------------------------------
58 // We use 1 bit after text field as an indicator to determine whether this child
59 // is an intermediate node or leaf node. If it is a leaf node, the sequence of
60 // texts visited so far from the start node to here can be returned as a valid
61 // suggestion to users with leaf_score.
62 //
63 // The size of score and address will be given in the first two bytes of the
64 // model file.
65 class OnDeviceHeadModel {
66  public:
67  // Gets top "max_num_matches_to_return" suggestions and their scores which
68  // matches given prefix.
69  static std::vector<std::pair<std::string, uint32_t>> GetSuggestionsForPrefix(
70  const std::string& model_filename,
71  const uint32_t max_num_matches_to_return,
72  const std::string& prefix);
73 };
