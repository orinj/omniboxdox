59  // Given a std::u16string in |term_string|, scans the history index and
60  // returns a vector with all scored, matching history items. The
61  // |term_string| is broken down into individual terms (words), each of which
62  // must occur in the candidate history item's URL or page title for the item
63  // to qualify; however, the terms do not necessarily have to be adjacent. We
64  // also allow breaking |term_string| at |cursor_position| (if
65  // set). Once we have a set of candidates, they are filtered to ensure
66  // that all |term_string| terms, as separated by whitespace and the
67  // cursor (if set), occur within the candidate's URL or page title.
68  // Scores are then calculated on no more than |kItemsToScoreLimit|
69  // candidates, as the scoring of such a large number of candidates may
70  // cause perceptible typing response delays in the omnibox. This is
71  // likely to occur for short omnibox terms such as 'h' and 'w' which
72  // will be found in nearly all history candidates. Results are sorted by
73  // descending score. The full results set (i.e. beyond the
74  // |kItemsToScoreLimit| limit) will be retained and used for subsequent calls
75  // to this function. In total, |max_matches| of items will be returned in the
76  // |ScoredHistoryMatches| vector.
77  ScoredHistoryMatches HistoryItemsForTerms(
78  std::u16string term_string,
79  size_t cursor_position,
80  size_t max_matches,

169  // Support caching of term results so that we can optimize searches which
170  // build upon a previous search. Each entry in this map represents one
171  // search term from the most recent search. For example, if the user had
172  // typed "google blog trans" and then typed an additional 'l' (at the end,
173  // of course) then there would be four items in the cache: 'blog', 'google',
174  // 'trans', and 'transl'. All would be marked as being in use except for the
175  // 'trans' item; its cached data would have been used when optimizing the
176  // construction of the search results candidates for 'transl' but then would
177  // no longer needed.
178  //
179  // Items stored in the search term cache. If a search term exactly matches one
180  // in the cache then we can quickly supply the proper |history_id_set_| (and
181  // marking the cache item as being |used_|. If we find a prefix for a search
182  // term in the cache (which is very likely to occur as the user types each
183  // term into the omnibox) then we can short-circuit the index search for those
184  // characters in the prefix by returning the |word_id_set|. In that case we do
185  // not mark the item as being |used_|.
186  struct SearchTermCacheItem {
187  SearchTermCacheItem(const WordIDSet& word_id_set,
188  const HistoryIDSet& history_id_set);
189  // Creates a cache item for a term which has no results.
190  SearchTermCacheItem();
191  SearchTermCacheItem(const SearchTermCacheItem& other);
192 
193  ~SearchTermCacheItem();
194 
195  // Estimates dynamic memory usage.
196  // See base/trace_event/memory_usage_estimator.h for more info.
197  size_t EstimateMemoryUsage() const;
198 
199  WordIDSet word_id_set_;
200  HistoryIDSet history_id_set_;
201  bool used_; // True if this item has been used for the current term search.
202  };
203  typedef std::map<std::u16string, SearchTermCacheItem> SearchTermCacheMap;
