113  // Adjusts the copied text before writing it to the clipboard. If the copied
114  // text is a URL with the scheme elided, this method reattaches the scheme.
115  // Copied text that looks like a search query will not be modified.
116  //
117  // |sel_min| gives the minimum of the selection, e.g. min(sel_start, sel_end).
118  // |text| is the currently selected text, and may be modified by this method.
119  // |url_from_text| is the GURL interpretation of the selected text, and may
120  // be used for drag-and-drop models or writing hyperlink data types to
121  // system clipboards.
122  //
123  // If the copied text is interpreted as a URL:
124  // - |write_url| is set to true.
125  // - |url_from_text| is set to the URL.
126  // - |text| is set to the URL's spec. The output will be pure ASCII and
127  // %-escaped, since canonical URLs are always encoded to ASCII.
128  //
129  // If the copied text is *NOT* interpreted as a URL:
130  // - |write_url| is set to false.
131  // - |url_from_text| may be modified, but might not contain a valid GURL.
132  // - |text| is full UTF-16 and not %-escaped. This is because we are not
133  // interpreting |text| as a URL, so we leave the Unicode characters as-is.
134  void AdjustTextForCopy(int sel_min,
135  std::u16string* text,
136  GURL* url_from_text,
137  bool* write_url);

220  // Asks the browser to load |match|. |index| is only used for logging, and
221  // can be kNoMatch if the popup was closed, or if none of the suggestions
222  // in the popup were used (in the unusual no-default-match case). In that
223  // case, an artificial result set with only |match| will be logged.
224  //
225  // OpenMatch() needs to know the original text that drove this action. If
226  // |pasted_text| is non-empty, this is a Paste-And-Go/Search action, and
227  // that's the relevant input text. Otherwise, the relevant input text is
228  // either the user text or the display URL, depending on if user input is
229  // in progress.
230  //
231  // |match| is passed by value for two reasons:
232  // (1) This function needs to modify |match|, so a const ref isn't
233  // appropriate. Callers don't actually care about the modifications, so a
234  // pointer isn't required.
235  // (2) The passed-in match is, on the caller side, typically coming from data
236  // associated with the popup. Since this call can close the popup, that
237  // could clear that data, leaving us with a pointer-to-garbage. So at
238  // some point someone needs to make a copy of the match anyway, to
239  // preserve it past the popup closure.
240  void OpenMatch(AutocompleteMatch match,
241  WindowOpenDisposition disposition,
242  const GURL& alternate_nav_url,
243  const std::u16string& pasted_text,
244  size_t index,

515  enum PasteState {
516  NONE, // Most recent edit was not a paste.
517  PASTING, // In the middle of doing a paste. We need this intermediate
518  // state because OnPaste() does the actual detection of
519  // paste, but OnAfterPossibleChange() has to update the
520  // paste state for every edit. If OnPaste() set the state
521  // directly to PASTED, OnAfterPossibleChange() wouldn't know
522  // whether that represented the current edit or a past one.
523  PASTED, // Most recent edit was a paste.
524  };
