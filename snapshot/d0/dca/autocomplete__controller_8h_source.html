38 // The AutocompleteController is the center of the autocomplete system. A
39 // class creates an instance of the controller, which in turn creates a set of
40 // AutocompleteProviders to serve it. The owning class can ask the controller
41 // to Start() a query; the controller in turn passes this call down to the
42 // providers, each of which keeps track of its own matches and whether it has
43 // finished processing the query. When a provider gets more matches or finishes
44 // processing, it notifies the controller, which merges the combined matches
45 // together and makes the result available to interested observers.
46 //
47 // The owner may also cancel the current query by calling Stop(), which the
48 // controller will in turn communicate to all the providers. No callbacks will
49 // happen after a request has been stopped.
50 //
51 // IMPORTANT: There is NO THREAD SAFETY built into this portion of the
52 // autocomplete system. All calls to and from the AutocompleteController should
53 // happen on the same thread. AutocompleteProviders are responsible for doing
54 // their own thread management when they need to return matches asynchronously.
55 //
56 // The coordinator for autocomplete queries, responsible for combining the
57 // matches from a series of providers into one AutocompleteResult.
58 class AutocompleteController : public AutocompleteProviderListener,
59  public base::trace_event::MemoryDumpProvider {

215  // Updates |result_| to reflect the current provider state and fires
216  // notifications. If |regenerate_result| then we clear the result
217  // so when we incorporate the current provider state we end up
218  // implicitly removing all expired matches. (Normally we allow
219  // matches from the previous result set carry over. These stale
220  // results may outrank legitimate matches from the current result
221  // set. Sometimes we just want the current matches; the easier way
222  // to do this is to throw everything out and reconstruct the result
223  // set from the providers' current data.)
224  // If |force_notify_default_match_changed|, we tell NotifyChanged
225  // the default match has changed even if it hasn't. This is
226  // necessary in some cases; for instance, if the user typed a new
227  // character, the edit model needs to repaint (highlighting changed)
228  // even if the default match didn't change.
