118  // Ensures that matches with headers, i.e., matches with a suggestion_group_id
119  // value, are grouped together at the bottom of result set based on their
120  // suggestion_group_id values and in the order the group IDs first appear.
121  // Certain types of remote zero-prefix matches need to appear under a header
122  // for transparency reasons. This information is sent to Chrome by the server.
123  // Also it is possible for zero-prefix matches from different providers (e.g.,
124  // local and remote) to mix and match. Hence, we group matches with the same
125  // headers and demote them to the bottom of the result set to ensure, one,
126  // matches without headers appear at the top of the result set, and two, there
127  // are no interleaving headers whether this is caused by bad server data or by
128  // mixing of local and remote zero-prefix suggestions.
129  // Note that prior to grouping and demoting the matches with headers, we strip
130  // all match group IDs that don't have an equivalent header string;
131  // essentially treating those matches as if they did not belong to any
132  // suggestion group.
133  // Called after matches are deduped and sorted and before they are culled.
134  void GroupAndDemoteMatchesWithHeaders();
